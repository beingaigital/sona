--------------------------------------------------------------------------------
1. 本次负责目标
--------------------------------------------------------------------------------
【对照升级计划任务 19】将 Sona 事件分析能力以 HTTP API 形式暴露，便于未来 Opinion System
等外部系统用统一 JSON 协议触发任务并获取 task_id、报告路径；同时保留 Streamlit 给人用。

--------------------------------------------------------------------------------
2. 完成内容
--------------------------------------------------------------------------------
- 使用 FastAPI 提供 GET /health、POST /v1/analyze-event、GET /v1/tasks/{task_id}、
  GET /v1/tasks/{task_id}/report（HTML 文件响应）。
- 事件分析路径与 Streamlit「事件模式」一致：route_query 判断历史数据、调用
  run_event_analysis_workflow；报告路径从会话消息中 report_html 工具消息解析。
- 通过 Typer 增加「sona serve」，内部用 uvicorn 启动，并将工作目录切到项目根，
  避免导入 api 包失败。
- 编写 docs/api_design.md，约定 JSON 字段与验收用 curl/PowerShell 命令。

--------------------------------------------------------------------------------
3. 验收方式（我们如何自证可用）
--------------------------------------------------------------------------------
- 启动 sona serve 后 /health 返回 200。
- /docs 可查看 OpenAPI 并试调接口。
- 完整 analyze-event 依赖模型与数据采集环境，未配齐时可能失败；接口层会返回
  failed 与 error 字段，不掩盖异常。

--------------------------------------------------------------------------------
4. 已知限制与后续可做
--------------------------------------------------------------------------------
- 任务状态仅存内存，进程重启后丢失；多 worker 不共享。
- analyze-event 当前为同步长请求，后续可改为异步队列 + 轮询。
- wiki、专题监测等端点按升级计划留作「计划中」，未在本次实现。

--------------------------------------------------------------------------------

  - Streamlit 方案：5 个页面模块（任务状态/新建/报告/检索/配置），侧边栏助手导航
  - 交付物：docs/gui_decision.md、sona_gui/ 目录代码
