# 交通舆情领域 Playbook

面向分析师与 Agent：在**有证据**的前提下，将交通类舆情拆成「事实链—情绪链—责任链—政策链」，再匹配回应动作。无证据处须标注「证据不足」，禁止补全通报细节或编造伤亡数字。

**材料边界**：行业规律、类型统计与对策归纳以本包 `materials/交通舆情课题摘录/` 为准；**具体事件的经过、定责、伤亡与官方通报原文**须单独取证（通报、裁判文书、权威媒体报道交叉验证），禁止仅用摘录稿替代新闻核查。

## 1. 领域边界

- **强相关**：轨道交通（含高铁、城际、市域铁路）运营与安全；12306/票务规则与公平感；民航运输、机场地面运行；公路交通事故与路权争议（含辅助驾驶伦理）；水路运输、港口、海事与危化；城市公交与枢纽接驳政策（含网约车、出租车）；快递物流与末端服务纠纷。
- **弱相关**：仅在车站、机舱、港区等为背景板的治安、性别对立等议题 — 若事件核心已偏离运输服务与安全，归相应社会议题域；本 Playbook 仅保留 **公共场所信息发布、疏散与联运衔接** 侧建议。

## 2. 高频主体

- 交通运输主管部门及地方交投、港航、邮政监管机构  
- 海事、应急管理、生态环境（涉危化与污染）、公安交管  
- 铁路局/车站段、航司、机场集团、公交企业、网约车平台  
- 港口、航运、船舶所有人/经营人，快递总部与加盟网点  
- 垂直交通自媒体、法律/科普博主、财经与国际政治账号（涉外海事）

## 3. 议题地图

- **轨道交通**：大面积晚点、设备故障、乘客冲突/霸座、票务与车厢服务（餐饮/标价）、防汛防震等不可抗力下列车调整。  
- **民航**：航班延误与取消、颠簸/备降、旅客突发疾病与空地联动、行李安检与枢纽网约车规则。  
- **公路与城市**：追尾/团雾连环撞、弱势交通参与者进入快速路、公交限速与安全整治、枢纽打车公平感。  
- **水路港口**：商渔船碰撞、危险品燃爆、航道拥堵谣言、桥梁防撞与次生议题链。  
- **物流**：丢件损毁、末端压榨、总部回应迟滞引发的「行业傲慢」叙事。  
- **政策与国际**：收费公路与春运等政策讨论的「知识漏斗」误读；高铁走出去、一带一路项目的境外框架战。

## 4. 独特传播机制

| 机制 | 说明 |
|------|------|
| 业态决定首发场 | 轨道/民航易在微博形成热搜与大 V 矩阵；公路更多下沉至地方抖音与交通广播；水路依赖航运垂直社群与专业自媒体。 |
| 算法二次扩散 | 头条系滞后微博约常见时间差，承担「地域兴趣」扩散；短视频多承接冲击画面与情绪剪辑。 |
| 议题链条外溢 | 铁路冲突类易沿「素质—执法—赔偿—制度信任」外扩；海事涉外事件易被财经/国际政治账号升格为地缘叙事。 |
| 技术性长尾 | 空难、危化原因调查周期长，知乎等社区承接「制度/技术」讨论，拉长生存周期。 |
| 服务类快消条件 | 单纯服务争议若配套 **12306/航司 APP/工单** 闭环响应，衰减更快；一旦卷入身份政治或「区别对待」，快消失效。 |

## 5. 风险模式（RiskPattern）

| 模式 | 触发信号 | 典型升级路径 |
|------|-----------|----------------|
| 事多情少 | 仅有程序性「情况通报」 | 「冷冰冰」标签 → 比照历史事件追责 → 质疑体制 |
| 定责模糊 | 辅助驾驶、多重责任主体、调查未终结 | 车企/驾驶人/道路管理互甩 → 阴谋论与技术谣言填空 |
| 公平感燃烧 | 价差、票务规则、外籍差别待遇等框架可被接入 | 个案 → 「规则歧视」→ 制度信任危机 |
| 枢纽规则误读 | 网约车禁限政策表述含糊 | 误读范围 → 出租车与平台对立 → 短视频情绪化对立 |
| 港城安全耦合 | 港区危化、浓烟可见度高 | 专业信息真空 → 全城恐慌与「毒性」谣言 |
| 物流总部失声 | 快递网点推诿、无工单编号 | 个案上升为「全行业不尊重消费者」 |
| 次生舆情 | 仅删帖或怼用户 | 从「事件本身」转向「态度与权力」 |

## 6. 回应策略（ResponseTactic）

1. **先时间线，后态度**：发布可追溯的关键节点（发现—处置—恢复通行/运行）；缺失节点主动说明「仍在调查」及下一次通报预期时间。  
2. **事心双解（事实—价值）**：在合法合规前提下，事实段之后呈现责任姿态与对受影响人群的关怀，避免只有流程复诵（对齐课题报告「事理有余、情理不足」的诊断）。  
3. **分层发布**：一线员工不单独承担定性责任；集团/监管统一口径；法务与宣传分轨，避免「法言法语」碾压共情。  
4. **业态补丁**：轨道侧重 **12306/车站广播/矩阵账号** 同步；民航侧重 **改签退票食宿** 清单；水路侧重 **海事+应急+生态** 分段背书；公路侧重 **警方首发+鉴定进度条**；枢纽政策侧重 **禁什么/不禁什么+替代出行地图**。  
5. **政策与谣言**：源头辟谣 + **误解对照表**（建议稿≠施行稿），减少只对线条「不实」的抽象否认。  
6. **平台协同**：对恶意剪辑提供时间戳对照素材（合规前提下）；涉外议题准备 **中英文事实页**，对冲模板化负面框架。  
7. **闭环披露**：整改台账（培训、设备更换、预案修订）可核查，减轻「通报即终点」的二次质疑。

## 7. 证据与合规要点

- 伤亡人数、事故性质、责任认定：以 **有权机关或授权主体** 通报为准；调查期内避免使用宣泄式定罪措辞，慎用「重大」「恶劣」等非授权定性标签。  
- 个人隐私与未成年人信息、行车记录仪/机舱争执音视频：**最小必要披露**，二次传播提示合法边界。  
- 危化与环境影响：未经采样与实验室结论，不猜测具体化学品危害结论；同步引导权威环境监测渠道。  
- 引用本库案例或摘录时，须回链 `registry.yaml` 所列路径及原始通报链接（若有）。

### 证据层级（取用顺序）

1. 监管/司法机关通报与调查报告（节选）  
2. 集团/运营主体经授权的官方声明（微博蓝 V、官网、新闻发布会）  
3. 行业技术标准与普法说明（辅助驾驶分级、海事避险规则等）  
4. 媒体报道（须交叉验证）；本包 `materials/` 摘录仅作模式归纳，不作单一事实源
